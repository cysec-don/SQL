🔧 Setup Guide

Get the CySec SQLi Training Lab running with Docker, or build from source.

🐳 Docker Quick-Start (Recommended)

The fastest way to launch the entire lab stack — application, MySQL database, and all dependencies — in isolated containers.

1. Clone the Repository

git clone https://github.com/cysecdon/cysec-sqli-lab.git
cd cysec-sqli-lab

2. Launch with Docker Compose

docker compose up -d

3. Initialize the Database

docker compose exec app python init_db.py

4. Access the Lab

Open your browser at: http://localhost:5000

5. Tear Down

docker compose down
docker compose down -v   # also remove volumes (resets database)
Tip: Docker Compose automatically maps ports, configures the MySQL connection, and seeds the database with challenge data. No manual configuration needed.

💻 Build from Source

For development or custom deployments, build the Docker image yourself.

1. Build the Docker Image

docker build -t cysec-sqli-lab .

2. Run the Container

docker run -d \
  --name cysec-lab \
  -p 5000:5000 \
  -e DB_HOST=mysql \
  -e DB_USER=sqli_user \
  -e DB_PASS=sqli_pass_2024 \
  -e DB_NAME=sqli_lab \
  cysec-sqli-lab

3. Start a MySQL Container

docker run -d \
  --name mysql \
  -e MYSQL_ROOT_PASSWORD=root_secret \
  -e MYSQL_DATABASE=sqli_lab \
  -e MYSQL_USER=sqli_user \
  -e MYSQL_PASSWORD=sqli_pass_2024 \
  -p 3306:3306 \
  mysql:8.0
Note: Make sure the application container can reach the MySQL container. Use --network or Docker Compose networking.

🖥 Manual Setup (Python + MySQL)

Run the lab directly on your machine without Docker.

Prerequisites

  • Python 3.9 or higher
  • MySQL 8.0+ (or MariaDB 10.5+)
  • pip and virtualenv

1. Create and Activate a Virtual Environment

cd cysec-sqli-lab
python3 -m venv venv
source venv/bin/activate      # Linux / macOS
# venv\Scripts\activate       # Windows

2. Install Python Dependencies

pip install -r requirements.txt

3. Set Up MySQL Database

# Log into MySQL as root
mysql -u root -p

# Create database and user
CREATE DATABASE sqli_lab CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'sqli_user'@'localhost' IDENTIFIED BY 'sqli_pass_2024';
GRANT ALL PRIVILEGES ON sqli_lab.* TO 'sqli_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

4. Configure Environment Variables

cp .env.example .env
# Edit .env with your database credentials:
#   DB_HOST=localhost
#   DB_USER=sqli_user
#   DB_PASS=sqli_pass_2024
#   DB_NAME=sqli_lab
#   SECRET_KEY=your-random-secret-key

5. Initialize the Database Schema

python init_db.py

6. Run the Application

# Development mode (auto-reload)
flask run --debug --host=0.0.0.0 --port=5000

# Production mode
gunicorn -w 4 -b 0.0.0.0:5000 app:app
Default Credentials: The lab seeds several accounts including admin:Ch@ll3ng3_P@ss! for testing authentication challenges. These are intentionally weak to aid learning.

🛠 Troubleshooting

Problem Solution
Can't connect to MySQL Verify MySQL is running and credentials match .env. Check with mysql -u sqli_user -p.
Port 5000 already in use Either stop the conflicting process or change the port: flask run -p 8080.
Module not found Ensure your virtual environment is activated and run pip install -r requirements.txt again.
Database tables missing Run python init_db.py to seed the database with challenge data.
Docker build fails Check Dockerfile syntax and ensure you have enough disk space. Try docker system prune.
