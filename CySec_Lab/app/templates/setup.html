🔧 Setup Guide

Get the CySec Don SQLi Training Lab running with Docker, or build from source.

🐳 Docker Compose (Recommended)

The fastest way to launch the entire lab stack — Flask app, MariaDB database, and phpMyAdmin — in isolated containers.

1. Clone the Repository

git clone https://github.com/cysec-don/SQL.git
cd SQL/CySec_Lab

2. Launch with Docker Compose

docker compose up -d --build

3. Wait for Services to be Healthy (~30 seconds)

docker compose ps

4. Access the Lab

  • Lab Application: http://localhost:5000
  • phpMyAdmin: http://localhost:8080

5. Tear Down

docker compose down
docker compose down -v   # also remove volumes (resets database)
Tip: Docker Compose automatically maps ports, configures the MariaDB connection, and seeds the database with challenge data. No manual configuration needed.

🐳 Docker Run (Standalone)

Build the image and run with a separate database container.

1. Start MariaDB

docker run -d --name cysec-db \
  -e MARIADB_ROOT_PASSWORD=root_password_change_me \
  -e MARIADB_DATABASE=cysec_lab \
  -e MARIADB_USER=cysec_lab \
  -e MARIADB_PASSWORD=cysec_lab_pass \
  -v $(pwd)/db/init.sql:/docker-entrypoint-initdb.d/01-init.sql:ro \
  -p 3306:3306 \
  mariadb:11.4

2. Wait for MariaDB to Initialize

sleep 15

3. Build and Start the Lab

docker build -t cysec-lab .
docker run -d --name cysec-app \
  --link cysec-db:db \
  -p 5000:5000 \
  cysec-lab
Note: Access the lab at http://localhost:5000. The database auto-initializes via init.sql.

💻 Build from Source

Run the lab directly on your machine without Docker using Python and MariaDB/MySQL.

Prerequisites

  • Python 3.10 or higher
  • MariaDB 10.6+ or MySQL 8.0+
  • pip and virtualenv

Option A: Automated Setup Script

cd SQL/CySec_Lab
bash scripts/build.sh

The script will check prerequisites, create a virtual environment, install dependencies, configure the database, import the schema, and create a .env file for you.

Option B: Manual Setup

1. Create and Activate a Virtual Environment

python3 -m venv venv
source venv/bin/activate      # Linux / macOS

2. Install Python Dependencies

pip install -r requirements.txt

3. Set Up MariaDB/MySQL Database

mysql -u root -p

CREATE DATABASE cysec_lab CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'cysec_lab'@'localhost' IDENTIFIED BY 'cysec_lab_pass';
GRANT ALL PRIVILEGES ON cysec_lab.* TO 'cysec_lab'@'localhost';
FLUSH PRIVILEGES;
EXIT;

4. Import the Schema

mysql -u cysec_lab -p'cysec_lab_pass' cysec_lab < db/init.sql

5. Configure Environment Variables

export FLASK_SECRET_KEY=$(python3 -c "import secrets; print(secrets.token_hex(32))")
export DB_HOST=localhost
export DB_PORT=3306
export DB_USER=cysec_lab
export DB_PASSWORD=cysec_lab_pass
export DB_NAME=cysec_lab
export SECURITY_LEVEL=low

6. Run the Application

python -m app.main
Default Credentials: All user accounts (admin, alice, bob, etc.) have the password password for lab convenience. See the README for the full list.

🔄 Reset Database

Reset all data to the initial state.

  • Docker: Visit http://localhost:5000/reset-db in your browser
  • Source: Re-run mysql -u cysec_lab -p'cysec_lab_pass' cysec_lab < db/init.sql
  • Docker Compose (nuclear): docker compose down -v then docker compose up -d

🛠 Troubleshooting

Problem Solution
Waiting for database... Ensure MariaDB is running: docker compose ps. The app retries for 60 seconds.
Port 5000 already in use Stop the conflicting process, or edit docker-compose.yml ports (e.g., 5001:5000).
Database connection refused Check DB_HOST matches container name (db for Docker, localhost for source).
Module not found Ensure virtual environment is activated and run pip install -r requirements.txt.
Database tables missing Visit /reset-db endpoint or re-import init.sql.
Docker build fails Check Dockerfile syntax and free disk space. Try docker system prune.
Blank page / 500 error Check logs: docker compose logs -f app.
